一、代码撤销
   1.若在使用git时 输入了一段错误的代码在没有提交时
        $ git checkout -- 文件名 将工作区的修改全部撤销与当前版本库内容保持一致
        !注意 撤销操作一定要加 -- 若不加 -- 该命令不是撤销而是切换分支
        撤销操作不能清除已经存放在暂存区的内容
   2.若已经通过add 将工作区的内容添加到暂存区,撤销添加到暂存区的修改
        $ git reset HEAD <filename> 撤销对指定工作区文件添加暂存区的操作
        再通过git checkout -- 文件名 撤销指定文件的内容
   3. 添加多个文件
        $ git add -A          将仓库中所有文件提交
        $ git add fileA fileB
        $ git add -U          将仓库中所有发生改变的文件提交
